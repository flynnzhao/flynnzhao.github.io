Hexo搭建Github静态博客

题记：
搭建本网站纯属一个巧合。在实验室做移动web开发，经常需要用到外网的环境做App测试。奈何实验室配置不给力，就想着自己折腾一个云个主机、申请个域名。想着很容易，但真正做起来才发现真特么麻烦。

首先在腾讯云上注册了.cn域名，买了个单核的云主机（最低配置，每个月￥65）。但是做域名解析时提示我必须先备案。后来查资料才知道：当域名要解析到内地服务器时，必须备案。所以硬着头皮，按照腾讯云提供的备案流程，签署相关文件，上传凭证，初审，拍照，邮寄材料等等,整套流程下来怎么着也得小1个月的时间（特别是提交管局，竟然提醒我要等20天左右，你懂的，废话就不多说了…）。

搭建个人博客的框架很多，比较了好多，最后一直在WordPress和Hexo上犹豫不决。首先尝试在云主机上搭建WordPress相关的环境配置，心情绝对是崩溃到不行，那个卡顿啊。因为云主机是单核的单核的单核的（1GB内存，50GB硬盘）。最后义无反顾的选择了Hexo的怀抱，不仅仅省了云主机的钱，相比WordPress的搭建真的太省劲了…

正文：

一、hexo是什么？

hexo是一款基于Node.js的静态博客框架。通俗点说就是一款基于node的静态博客网站生成器。使用起来相当的简单，它生成静态网页的速度也是快的像风一样。

hexo常用命令行：

npm install hexo -g #安装hexo
hexo init #初始化
npm install #安装依赖库

hexo g #生成静态页面至public目录
hexo s #开启预览访问端口（默认端口4000，’ctrl + c’关闭server）
hexo deploy #将.deploy目录部署到GitHub

查看全部命令行，输入hexo help命令，如下图

二、hexo安装

1、环境配置
顺着点下一步，除了安装路径以外其余设置保持默认安装配置即可。
安装Git和node.js。申请github账户，用来托管。

2、博客搭建

  • 鼠标右键任意地方，选“Git Bush Here” 输入代码安装Hexo

    $ npm install -g hexo


    出现上图，说明hexo安装成功。

  • 在电脑D盘新建一个文件夹（用于存放生成的hexo相关文件），在该文件夹内右键“Git
    Bush Here”输入以下两条命令代码

    $ hexo init

    $ npm install

    至此，hexo依赖包安装结束。

  • 生成博客的静态网页，输入一下两条命令代码

    $ hexo g

    $ hexo s

    如果看到如图所示的最后一行提示，恭喜你，博客的雏形已经搭建完成。打开浏览器访问：http://localhost:4000/

    这是hexo默认样式的apparence 。hexo官网有很多漂亮的主题，也有很多个人开发的第三方主题，可根据自己的喜好去调整。具体的怎么更换这里就不再赘述。

三、github部署

  1. 新建repository

    new一个与你用户名对应的public权限的repository，仓库名必须为【your_user_name.github.io】,就是你注册github账户名。

  2. 部署静态博客
    打开本地hexo下的配置文件_config.yml，并配置好部署信息。


    部署信息如下
    deploy:
    type: git
    repository:https://github.com/your_user_name/your_user_name.github.io.git
    branch: master

    配置好_config.yml并保存后，执行以下命令部署到Github上。

    $ npm install hexo-deployer-git –save

    $ hexo g

    $ hexo d

    执行“$ hexo d ”命令时如果一切顺利，过程中会弹出两个窗口，一个是提醒输入github账户用户名，一个是输入github账户密码。

  1. 成功部署后后浏览器输入your_user_name.github.io，是不是可以访问了，有没有很开心！！！

四、绑定个人独立域名

  1. 绑定域名之前，先要做好域名解析。
    在哪个平台购买的域名，就在哪儿做CNAME。因为这没有独立的云主机，所以做域名解析的时候只需把个人独立域名解析到github下的个人域名“your_user_name.github.io”即可。例如博主在腾讯云域名注册平台申请的域名，在域名管理里可以这样做解析，如图

  2. github上打开上文建立的repository，new file一个CNAME文件，输入您的个人域名。例如：labcode.cn 。 保存后退出。此时浏览器输入个人域名，奇迹就发生了。

结束语：码文不易，转载前请注明出处，谢谢！

什么是 Cmd Markdown

您可以使用很多工具书写 Markdown，但是 Cmd Markdown 是这个星球上我们已知的、最好的 Markdown 工具——没有之一 ：）因为深信文字的力量，所以我们和你一样，对流畅书写，分享思想和知识，以及阅读体验有极致的追求，我们把对于这些诉求的回应整合在 Cmd Markdown，并且一次，两次，三次，乃至无数次地提升这个工具的体验，最终将它演化成一个 编辑/发布/阅读 Markdown 的在线平台——您可以在任何地方，任何系统/设备上管理这里的文字。

1. 实时同步预览

我们将 Cmd Markdown 的主界面一分为二，左边为编辑区，右边为预览区，在编辑区的操作会实时地渲染到预览区方便查看最终的版面效果，并且如果你在其中一个区拖动滚动条，我们有一个巧妙的算法把另一个区的滚动条同步到等价的位置，超酷！

2. 编辑工具栏

也许您还是一个 Markdown 语法的新手，在您完全熟悉它之前，我们在 编辑区 的顶部放置了一个如下图所示的工具栏，您可以使用鼠标在工具栏上调整格式，不过我们仍旧鼓励你使用键盘标记格式，提高书写的流畅度。

3. 编辑模式

完全心无旁骛的方式编辑文字：点击 编辑工具栏 最右测的拉伸按钮或者按下 Ctrl + M，将 Cmd Markdown 切换到独立的编辑模式，这是一个极度简洁的写作环境，所有可能会引起分心的元素都已经被挪除，超清爽！

4. 实时的云端文稿

为了保障数据安全，Cmd Markdown 会将您每一次击键的内容保存至云端，同时在 编辑工具栏 的最右侧提示 已保存 的字样。无需担心浏览器崩溃，机器掉电或者地震，海啸——在编辑的过程中随时关闭浏览器或者机器，下一次回到 Cmd Markdown 的时候继续写作。

5. 离线模式

在网络环境不稳定的情况下记录文字一样很安全！在您写作的时候，如果电脑突然失去网络连接，Cmd Markdown 会智能切换至离线模式，将您后续键入的文字保存在本地，直到网络恢复再将他们传送至云端，即使在网络恢复前关闭浏览器或者电脑，一样没有问题，等到下次开启 Cmd Markdown 的时候，她会提醒您将离线保存的文字传送至云端。简而言之，我们尽最大的努力保障您文字的安全。
